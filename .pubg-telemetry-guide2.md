# 📊 BGMS PUBG 텔레메트리 전술 분석 가이드 (V8.2)

이 문서는 BGMS 프로젝트에서 PUBG API 및 텔레메트리 데이터를 수집하여 고정밀 전술 지표를 산출하고, 이를 기반으로 AI 코칭 리포트를 생성하는 전체 아키텍처와 로직을 설명합니다. (V8.2 공간 지능 및 트레이드 로직 통합 업데이트 완료)

---

## 1. 분석 파이프라인 (Data Pipeline)

1. **Match API (`/api/pubg/match`)**:
   - PUBG API에서 매치 상세 및 텔레메트리 자산 URL을 가져옵니다.
   - **Master Cache (V16)**: 원본 텔레메트리(수십 MB)를 전술 분석에 필요한 핵심 이벤트만 남긴 '다이어트' 데이터로 변환하여 `match_master_telemetry` 테이블에 저장합니다. (V16부터 `LogPlayerCreate` 포함)
   - **Tactical Engine (V8.2)**: 
     - **노이즈 필터링**: 비행기 경로 및 낙하 단계의 무의미한 위치 데이터를 제거(Z < 10m 또는 120초 이후)하여 지상 전술 데이터만 추출합니다.
     - **전술 지표 산출**: 플레이어별 1:1 교전, 백업 속도, 고립 지수 등을 계산하여 `processed_match_telemetry`에 최종 결과를 저장합니다.

2. **AI Analyze API (`/api/pubg/ai-analyze`)**: [V5.1 안정화]
   - 매치 전술 지표를 Gemini 1.5/3.1 AI에게 전달하여 코칭 토론을 생성합니다.
   - **JSON Integrity**: 템플릿 리터럴 구문 최적화를 통해 AI 응답의 JSON 파싱 안정성을 극대화했습니다.

## 2. 핵심 전술 지표 정의 (Core Metrics)

### 1) 선제 타격 효율 (Initiative Rate)
- **정의**: 교전 발생 시 유저가 적보다 먼저 유효 타격을 가해 주도권을 잡은 비율.
- **V5.1 개선**: 시스템 플레이어(환경사, 자가부활 등)에 의한 데미지를 제외하여 순수 교전 지표의 정확도를 높였습니다.

### 2) 통합 전술 지원 (Tactical Support)
- **견제 사격 (Suppression)**: 아군 기절 후 120초 내 적에게 가한 압박.
- **연막 세이브 (Smoke Support)**: 기절 지점 주변 연막 전개 여부.
- **전술적 가치**: 직접 부활 횟수가 적더라도 견제/연막 지표가 높으면 '전술적 기여'로 인정.

### 3) 킬 기여도 유형 (Kill Contribution) [V5.0 유지]
- **Solo Kill**: 적 체력의 70% 이상을 직접 깎고 최종 사살까지 성공. (개인 무력 지표)
- **Cleanup**: 팀원이 70% 미만의 데미지를 준 적을 마무리하거나, 내가 주도했으나 팀원이 마무리한 경우.
- **Other**: 어시스트성 킬 또는 불특정 상황에서의 처치.

### 4) 전술적 몰살 (Enemy Team Wipe) [V5.0 유지]
- **정의**: 적 팀이 전멸했을 때, 내가 해당 팀 인원을 한 명이라도 사살했거나 마지막 인원을 처치한 경우.
- **의미**: 팀 단위 교전의 종결 및 결정적 참여 지표.

### 5) 교전 반응 속도 (Backup/Trade Latency) [V8.2 고도화]
- **Trade Latency (V8.2)**: 팀원이 기절한 시점부터 내가 해당 적을 **기절시키거나 사살(Finishing)**하여 상황을 반전시키기까지 걸린 시간. 단순 타격이 아닌 '제압 성공' 시에만 수치를 산정합니다.
- **Backup Latency**: 팀원이 기절한 시점부터 나의 첫 번째 유효 타격까지 걸린 시간 (상위권 평균 1.8초 미만).
- **Counter-Attack Latency**: 내가 데미지를 입은 후 적에게 반격 데미지를 입히기까지 걸린 시간.

### 6) 공간 지능 분석 (Spatial Intelligence / Isolation Index) [V8.1+ 신설]
- **고립 지수 (Isolation Index)**: 교전 발생 시 아군과의 평균 거리 및 상대적 안전도를 수치화한 지표.
- **측정 기준**: 아군 거리(Min Dist), 고도 일치성(Height Diff), 십자포화 노출(Crossfire Detection)을 종합 판정합니다.
- **데이터 정밀도**: 비행기/낙하 단계를 제외한 실제 지상 데이터를 기반으로 합니다.

### 6) 골든 타임 데미지 (Golden Time Damage) [복구]
- **Early (0-5분)**: 낙하 및 초반 교전 능력.
- **Mid1 (5-15분)**: 서클 이동 초반 운영 능력.
- **Mid2 (15-25분)**: 중후반 자기장 압축 구간 운영 능력.
- **Late (25분+)**: 최종 자기장 엔딩에서의 집중력.

### 7) 자기장 적응력 (Zone Adaptation) [V5.1 신설 → V8.3 명칭 수정]
- **✅ 공식 PUBG API 기준 명칭**:
  - `safetyZone` (safetyZoneRadius/Position) = **White Zone (안전구역, 흰색 원)** — 다음 수축될 안전 지점
  - `poisonGasWarning` (poisonGasWarningRadius/Position) = **Blue Zone (자기장, 파란색 원)** — 현재 자기장 경계
- **isZoneMoving 로직**: `blueRadius > whiteRadius * 1.05` (자기장이 안전구역보다 클 때 = 수축 중)
- **시각화 기술**: Canvas 레이어에 **선형 보간(Linear Interpolation)** 기술을 적용하여 부드러운 이동 구현.

## 3. AI 코칭 및 판정 논리 (AI Debate Strategy)

### 1) 페르소나 설정
- **KIND COACH**: 긍정적 지표 우선 탐색 및 상황적 맥락 옹호.
- **SPICY BOMBER**: 팩트 중심 독설 및 벤치마크 기반 공격.

### 2) 판정 가이드라인 [복구/보완]
- **Apple-to-Apple**: 유저 수치가 벤치마크 이상이면 `KIND COACH` 우세 판정.
- **상황적 옹호**: 연막/견제 수치가 높으면 직접 부활 미달을 비난하지 못하도록 프롬프트 제약 적용.
- **독백 금지**: 서로의 의견을 직접 인용하며 반박하는 '대화형 토론' 구조 유지.

## 4. 기술 사양 (Technical Specifications)

- **Telemetry Version**: 16 (LogPlayerCreate 이벤트를 통한 초기 생존 상태 추적 보장)
- **Engine Version**: 8.2 (트레이드 로직 및 공간 필터링 최신화)
- **Coordinates Inversion**: `8192 - y` 공식 적용 (맵 좌표계 보정).
- **Scale Normalization**: 텔레메트리 원본 좌표(cm)를 분석 엔진에서 **미터(m)** 단위로 변환하여 3D 거리 계산.
- **Smooth VFX**: Canvas 기반의 60fps 자기장 렌더링 및 보간 적용.
- **Data Diet**: 분석 불필요 이벤트를 90% 제거하여 최적화.

## 📚 Appendix: Official PUBG Telemetry Reference

이 섹션은 향후 전술 분석 로직 변경 및 데이터 확장을 위해 [PUBG Official Documentation](https://documentation.pubg.com/en/telemetry-events.html)의 핵심 내용을 요약한 것입니다.

### 1) 공통 이벤트 필드 (Common Fields)
모든 텔레메트리 이벤트는 아래 필드를 포함합니다.
- `_D`: 이벤트 발생 시간 (string, ISO 8601)
- `_T`: 이벤트 타입 이름 (string, e.g. "LogPlayerKill")
- `common`: { isGame: number } - 게임 진행 상태 (0: 매치 전, 0.1: 비행기, 0.5: 게임 중, 1.0: 게임 종료)

### 2) 핵심 전술 분석 이벤트 (Key Events)

#### [LogPlayerKillV2] - 킬/사망 정밀 분석
```json
{
  "attackId": int,
  "dBNOId": int,
  "victim": {Character},
  "dBNOMaker": {Character},      // 기절시킨 사람 (중요: 킬러와 다를 수 있음)
  "finisher": {Character},       // 확킬낸 사람
  "killer": {Character},         // 최종 킬 판정자
  "assists_AccountId": [string], // 어시스트 목록
  "isSuicide": bool
}
```

#### [LogPlayerTakeDamage] - 교전 및 선제 타격 분석
```json
{
  "attackId": int,
  "attacker": {Character},
  "victim": {Character},
  "damageTypeCategory": string,
  "damageReason": string,
  "damage": number,              // 실제 체력 감소량 (1.0 = 1 health)
  "damageCauserName": string
}
```

#### [LogGameStatePeriodic] - 자기장 및 경기 상태 업데이트
```json
{
  "gameState": {
    "safetyZonePosition": {Location},       // ✅ White Zone(안전구역/흰색 원) 중심 — 다음 수축 목표 지점
    "safetyZoneRadius": number,             // ✅ White Zone 반지름
    "poisonGasWarningPosition": {Location}, // ✅ Blue Zone(자기장/파란색 원) 중심 — 현재 자기장 경계
    "poisonGasWarningRadius": number,       // ✅ Blue Zone 반지름
    "numAlivePlayers": int,
    "numAliveTeams": int
  }
}
```
> ⚠️ 주의: 이전 가이드 버전에서는 safetyZone을 Blue(자기장), poisonGasWarning을 White(안전구역)로 표기하여 오류가 있었습니다.
> 코드베이스 전체가 V8.3에서 수정되었습니다 (telemetry/route.ts, match/route.ts, extract_bluezone.ts).


#### [LogPlayerUseThrowable] - 유틸리티 지원 분석
```json
{
  "attacker": {Character},
  "attackType": "Weapon",
  "weapon": { "itemId": "Item_Weapon_SmokeBomb_C", ... }
}
```

#### [LogPlayerPosition] - 실시간 고립도 측정
- **V8.1 필터**: `e.character.loc.z < 10` (지면) 또는 `elapsed > 120` (낙하 종료) 조건에서만 샘플링.
- **샘플링**: 팀원은 전량 수집, 적군은 1/10 확률로 샘플링하여 성능 최적화.

#### [LogPlayerCreate] - 생존 상태 초기화
- 매치 시작 시 플레이어의 초기 생존 여부를 명시적으로 설정하여 누락된 이벤트로 인한 계산 오류 방지.

### 3) 맵 좌표 보정 (Coordinate System)
- **Scale**: 텔레메트리 좌표는 맵 실제 크기(예: 800,000) 기준으로 전달됨.
- **Inversion**: 프론트엔드 렌더링 시 `8192 - y` (또는 전체 높이 - y) 보정이 필요할 수 있음.
- **Normalization**: BGMS 내부에서는 모든 맵 좌표를 0~8192 스케일로 정규화하여 처리함.
- **Tactical Scale**: 거리 계산 시에는 1/100을 적용하여 미터(m) 단위로 환산함.
