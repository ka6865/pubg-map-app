# 📖 BGMS AI 전략 컨텍스트 및 아키텍처 가이드 (v2026.04.29 최신판)

이 문서는 AI와 개발자가 BGMS 프로젝트의 모든 구성 요소를 완벽하게 이해하고, 유지보수 시 정확한 위치에서 작업할 수 있도록 모든 폴더와 파일의 역할을 초정밀하게 기록합니다.

---

## 🛠️ 기술 스택 (Tech Stack)

- **핵심 프레임워크**: Next.js 16.1.1 (App Router, Turbopack 최적화)
- **프론트엔드**: React 19.2.3 (React Compiler), Leaflet (지도), Lucide React (아이콘)
- **스타일링**: Tailwind CSS v4 (CSS 변수 기반 테마 시스템)
- **백엔드/인프라**: Supabase (Database, Auth, Realtime, Storage)
- **AI 서비스**: AI SDK (Vercel), Google Generative AI (Gemini 3.1 Flash / 3 Pro)
- **모바일 환경**: Capacitor v8 (하이브리드 앱), 레이아웃 표준 준수

---

## 📁 디렉토리 및 파일 상세 매핑 (Detailed Mapping)

### 1. `app/` - 프로젝트의 심장부 (Routing & Pages)
- `layout.tsx`: 전역 폰트(Inter), 테마, SEO 기본 설정 및 `AuthProvider` 래핑.
- `page.tsx`: 메인 홈 페이지. 맵 선택 및 최근 매치 요약 대시보드 진입점.
- `globals.css`: Tailwind v4 레이어 설정 및 전역 CSS 변수(Colors, Transitions).
- `manifest.ts`: PWA/모바일 설치를 위한 앱 메타데이터 설정.
- `robots.ts` / `sitemap.ts`: 검색 엔진 최적화(SEO)를 위한 크롤링 가이드.

- **주요 페이지 그룹**
    - `maps/`: 특정 맵(에란겔, 론도 등)의 전술 지도 페이지 (`[mapId]/page.tsx`).
    - `stats/`: 플레이어 전적 검색 및 분석 결과 표시 (`[platform]/[nickname]/page.tsx`).
    - `backpack/`: 아이템별 공간 효율성 및 파밍 최적화 가이드.
    - `weapons/`: 무기 스탯 및 부착물 조합 분석.
    - `board/`: 커뮤니티 게시판 (목록, 상세, 작성).
    - `admin/`: 관리자 대시보드 및 맵/게임 데이터 설정.
    - `mypage/`: 사용자 프로필 및 개인 설정.
    - `map-editor/`: 맵 마커 및 카테고리 편집 도구.
    - `auth/`: 인증 관련 콜백 처리.

- **`actions/` (Server Actions)**
    - `map-settings.ts`: 맵 카테고리 및 아이콘 속성 DB 연동 로직.

- **`api/` (Backend Endpoints)**
    - `pubg/ai-summary/`: 텔레메트리 기반 전체 요약 및 전술 배지 생성.
    - `pubg/ai-analyze/`: 개별 교전 상황 분석 및 코칭 피드백.
    - `pubg/match/`: PUBG API 매치 상세 정보 수집 및 DB 캐싱.
    - `pubg/telemetry/`: 텔레메트리 데이터 페칭 및 가공.
    - `pubg/hotdrop/`: 실시간/통계 기반 핫드랍 밀집도 데이터 제공.
    - `admin/`: 데이터 승인/거절, 패치노트 동기화, 게임 데이터 관리.
    - `cron/`: 핫드랍 업데이트 및 패치노트 자동화 예약 작업.

### 2. `components/` - 재사용 가능한 UI 블록
- **`map/` (Tactical Map Engine)**
    - `MapShell.tsx`: Leaflet 인스턴스화, 줌/센터 관리.
    - `MapView.tsx`: 맵 타일 레이어 및 마커 클러스터링 실구현.
    - `TelemetryPlayer.tsx`: 매치 로그 기반 플레이어 이동 경로 재생기.
    - `HotDropLayer.tsx`: 지도 위 핫드랍 열지도 오버레이.
    - `KillFeed.tsx`: 매치 타임라인별 킬 로그 시각화.
    - **`telemetry/`**: 텔레메트리 전용 렌더링 레이어
        - `TelemetryCanvasLayer.tsx`: 대량의 이동/교전 데이터를 Canvas로 고성능 렌더링.
        - `CombatRenderer.tsx` / `ShotRenderer.tsx`: 교전 및 사격 위치 시각화.
        - `PlayerMarkerRenderer.tsx` / `PlayerPathRenderer.tsx`: 플레이어 아이콘 및 경로 렌더링.

- **`stat/` (Analysis UI)**
    - `MatchCard.tsx`: 매치별 분석 결과 요약 카드 (V6.1 Space-Aware 반영).
    - `RecentAISummary.tsx`: 최근 10매치 통합 분석 AI 리포트.
    - `IsolationRadar.tsx` / `SpiderChart.tsx`: 전술적 고립도 및 육각형 스탯 시각화.
    - `MiniStatWidget.tsx`: 대시보드용 간이 전적 위젯.

- **`common/` & `layout/`**
    - `GlobalHeader.tsx` / `BottomNav.tsx`: 전역 내비게이션 및 모바일 툴바.
    - `Sidebar.tsx`: 지도 필터 및 카테고리 선택.
    - `Footer.tsx`: 사이트 정보 및 하단 영역.

### 3. `hooks/` - 비즈니스 로직 캡슐화
- `useTelemetry.ts`: [핵심] PUBG 텔레메트리 JSON 파싱 및 공간 분석(Isolation Index 등) 로직.
- `useMapData.ts`: 특정 맵의 제보 데이터 및 고정 마커 관리.
- `useMapSettings.ts`: 실시간 맵 설정(아이콘, 색상 등) 동기화.
- `useRealtimeToast.ts`: 실시간 알림 및 토스트 메시지 처리.

### 4. `lib/` & `utils/` - 핵심 설정 및 도구
- `lib/map_config.ts`: **[필독]** 맵별 좌표 변환 공식(론도 8192-y 등) 및 SVG 라이브러리.
- `lib/seo-config.ts`: 페이지별 동적 Metadata 설정.
- `lib/supabase-middleware.ts`: 세션 유효성 체크 미들웨어.
- `utils/supabase/`: 서버/클라이언트 환경별 Supabase 인스턴스 생성 도구.

## 🔧 주요 유지보수 워크플로우 (Core Workflows)

1. **공간 인지 전술 분석 (Space-Aware Analysis)**
    - `useTelemetry.ts`에서 거리, 무기 위협도, 교전 각도를 기반으로 3D Isolation Index 계산.
    - Z축 높이차를 이용한 수직 전술 유리성 판단 로직 준수.
2. **고성능 텔레메트리 렌더링**
    - `TelemetryCanvasLayer.tsx`를 통해 대량의 이벤트 데이터를 프레임 드랍 없이 재생.
    - `Canvas API`를 사용하여 블루존(자기장) 및 이동 경로를 부드럽게 보간(Interpolation).
3. **AI 코칭 엔진 (V6+)**
    - `ai-summary` 및 `ai-analyze` API를 통해 'Tough-love' 방식의 데이터 기반 피드백 생성.
    - 다중 페르소나(분석 전문가, 친절한 코치) 간의 토론 형식 리포트 지원.

## ⚠️ 무조건 지켜야 할 핵심 규칙 (Critical Rules)

1. **호출부-선언부 동기화 및 에러 보완**:
    - 함수 수정 시 해당 함수를 사용하는 모든 지점의 인자 타입과 개수를 반드시 재검토할 것.
    - **에러 보완**: 컴포넌트나 함수 사용 시 Import 누락으로 인한 런타임 에러가 발생하지 않도록 상시 검토할 것.
2. **텔레메트리 필드 준수**: `.pubg-telemetry-guide.md`를 참조하여 `maker`(기절), `killer`(확킬), `reviver`(부활) 필드를 정확히 구분하여 사용.
3. **Clean Code & No Dummy**: 
    - 하드코딩된 더미 데이터는 즉시 Supabase Actions로 교체.
    - 사용하지 않는 `console.log`, `import`, 주석 처리된 코드는 발견 즉시 삭제.
4. **RSC & Client 분리**: 모든 페이지는 기본적으로 Server Component(RSC)로 작성하며, 인터랙션이 필요한 부분만 Client Component로 분리하여 성능 최적화.
5. **보안 및 권한**: 어드민 전용 기능(`admin/`)은 반드시 서버 측에서 세션과 역할을 검증(`profile.role === 'admin'`).
6. **성능 및 SEO 최적화**:
    - 이미지 사용 시 `next/image`의 `priority` 속성을 활용하여 LCP(Largest Contentful Paint) 최적화.
    - `Metadata API`를 통해 동적 키워드(예: '론도 비밀의 열쇠', '에란겔 비밀의 열쇠' 등)를 삽입하여 검색 노출 극대화.
