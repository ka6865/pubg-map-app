# 📖 BGMS AI 전략 컨텍스트 및 아키텍처 가이드

이 문서는 AI(및 개발자)가 BGMS 프로젝트의 구조를 빠르게 이해하고, 유지보수 시 정확한 위치에서 작업을 수행할 수 있도록 돕기 위해 작성되었습니다.

---

## 🛠️ 기술 스택 요약 (Tech Stack)

- **Framework**: Next.js 16.1.1 (App Router, Turbopack 최적화)
- **Library**: React 19.2.3, Leaflet, Lucide React
- **Styling**: Tailwind CSS v4 (강력한 컨테이너 쿼리 및 유틸리티 활용)
- **Database / Auth**: Supabase (PostgreSQL)
- **AI Integration**: Google Generative AI (Gemini 2.0 Flash)
- **Mobile Hybrid**: Capacitor v8 (iOS 지원)

---

## 📁 디렉토리 구조 및 역할 (Directory Mapping)

### 1. `app/` (Routing & Entry Points)
Next.js의 App Router 기반으로 페이지 경로와 서버 측 엔트리를 담당합니다.
- `app/maps/`: 7개 맵(에란겔, 미라마 등)의 인터랙티브 상세 페이지.
- `app/stats/`: 플레이어 통계 검색 및 시각화 대시보드.
- `app/board/`: 커뮤니티 게시판 (상세 페이지, 글쓰기 포함).
- `app/api/`: 백엔드 서버리스 함수 폴더.
    - `api/pubg/`: PUBG API 호출 및 텔레메트리 파싱 로직.
    - `api/cron/`: 패치노트 자동 수집 및 AI 요약 스케줄러.
    - `api/admin/`: 게시글 승인/반려 등 관리용 기능.
- `app/mypage/`: 사용자 프로필 및 활동 내역.
- `app/login/`: Supabase Auth 기반 소셜 로그인 처리.

### 2. `components/` (UI Components)
도메인별로 세분화된 재사용 가능한 컴포넌트들입니다.
- `components/map/`: 지도 렌더러, 텔레메트리 레이어, 마커 시스템.
- `components/board/`: 게시판 리스트, 댓글창, 글쓰기 폼.
- `components/stat/`: 매치 카드, 플레이어 성장 그래프, AI 분석 보고서 UI.
- `components/common/`: 하단 네비게이션(`BottomNav`), 모바일 전용 메뉴 등 공통 요소.
- `components/admin/`: 관리 페이지 전용 대시보드 컴포넌트.

### 3. `hooks/` (Custom React Hooks)
복잡한 상태 로직을 캡슐화합니다.
- `useTelemetry.ts`: 대용량 게임 매치 로그를 파싱하여 지도에 애니메이션으로 투사하는 핵심 로직.
- `useMapData.ts`: 맵별 마커 위치 및 필터링 상태 관리.

### 4. `lib/` & `utils/` (Core Libraries & Utilities)
외부 서비스 설정 및 저수준 헬퍼 함수입니다.
- `lib/supabase.ts`: Supabase 클라이언트 싱글톤 설정.
- `lib/map_config.ts`: 맵별 좌표 및 줌 레벨 오프셋 값 관리.
- `utils/supabase/`: 서버/클라이언트 환경별 Supabase 인스턴스 생성기.

### 5. `types/` (TypeScript Definitions)
전역 타입 안정성을 보장합니다.
- `game-data.ts`: 아이템, 무기, 차량 등 인게임 데이터 타입.
- `telemetry.ts`: PUBG API 텔레메트리 로그 표준 포맷 정의.

### 6. `scripts/` & `public/`
- `scripts/build-app.sh`: 모바일 앱 배포를 위한 정적 내보내기(Static Export) 및 파일 전처리 스크립트.
- `public/`: 맵 타일 이미지, 자산(Assets) 저장소.

## 🔧 유지보수 가이드 (Maintenance Guide)

### 새로운 맵을 추가하거나 수정하고 싶다면?
1. `lib/map_config.ts`에서 해당 맵의 좌표 시스템과 줌 범위를 정의합니다.
2. `public/` 디렉토리에 맵 타일 이미지를 업로드합니다.
3. `app/maps/[mapId]/` 페이지에서 데이터가 정상적으로 매핑되는지 확인합니다.

### AI 분석 로직이나 프롬프트를 수정하고 싶다면?
1. `app/api/pubg/ai-analyze/route.ts` 또는 `ai-summary/route.ts` 파일을 확인합니다.
2. `Google Generative AI` 설정값과 시스템 프롬프트를 조정합니다.

### 모바일 UI의 레이아웃이 깨진다면?
1. `app/globals.css`의 Tailwind v4 설정을 확인합니다.
2. `components/common/BottomNav.tsx`와 상단 헤더 컴포넌트의 `safe-area` 패딩 설정을 점검합니다.

### 새로운 게시판 기능을 추가하려면?
1. `app/board/` 에 새 경로를 추가합니다.
2. `components/board/` 에서 필요한 UI 요소를 신규 생성합니다.
3. `types/board.ts`에 필요한 데이터 인터페이스를 업데이트합니다.

## ⚠️ 주의사항
- **RSC 우선**: 모든 페이지는 기본적으로 Server Components로 구성하여 번들 크기를 최소화하십시오. 상호작용이 필요한 경우에만 'use client'를 사용합니다.
- **모바일 지원**: Capacitor를 통한 하이브리드 앱을 지원하므로, `window`나 `document` 객체 접근 시 플랫폼 체크가 필요할 수 있습니다.
